渗透

原理

劫持sudo命令（触发反向连接）

测试

测试机先启动监听（nc -lvp 4444），目标机执行sudo ls，若反向连接成功，测试机可获得目标机的 Shell。

清理方案

优势（适合场景）

  • 隐蔽性中等：日常使用中，用户执行ls/cd等常用命令时，不会察觉别名背后的额外逻辑（除非主动检查别名）；
  • 实施成本低：无需编译恶意程序，仅通过修改 Shell 配置文件即可实现，适合快速植入；
  • 针对性强：可针对特定用户（普通用户 /root）配置，仅影响目标用户的 Shell 环境；
  • 功能灵活：可实现指令记录（日志）、环境劫持（如修改PATH）、触发特定动作（如反向连接）等。

局限性（风险点）

  • 依赖 Shell 环境：仅对配置了别名的 Shell（如bash/zsh）生效，切换 Shell（如从bash到sh）或通过非交互 Shell 执行命令时会失效；
  • 易被检测：通过alias命令可直接查看所有别名，检查~/.bashrc//etc/bashrc等配置文件也能发现异常；
  • 权限依赖：若要影响所有用户（全局别名），需root权限；仅影响当前用户，普通权限即可；
  • 非持久化（部分场景）：若用户删除别名或重建 Shell 配置文件，后门会失效（需结合其他持久化手段，如定时任务恢复配置）。

结论

可行，但仅适合短期、轻量级的指令劫持或日志记录场景，不适合高隐蔽性、长期持久化需求（需搭配其他技术，如LD_PRELOAD劫持、内核模块等）。在合法测试中，常用于验证目标系统对 “命令别名劫持” 这类基础攻击的防御能力。
